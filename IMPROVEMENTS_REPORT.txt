חלק א': תיקונים קלים (Bug Fixes) – סיכון אפס
═══════════════════════════════════════════════════════════════════════════════

1.1 כפתור החיפוש – searchFunction() חסר
   בעיה: כפתור "חפש" קורא ל-searchFunction() שלא מוגדר ב-Main Menu.js.
   פתרון: הוספת פונקציה ב-Main Menu.js:
   
   function searchFunction() {
       const input = document.getElementById('search');
       const query = input.value.toLowerCase().trim();
       if (!query) return;
       // שימוש באותה לוגיקה שכבר קיימת ב-addEventListener
       // (העתקת הלוגיקה מהאירוע input)
   }
   
   השפעה: כפתור "חפש" יעבוד גם בלחיצה, לא רק בהקלדה. אין שינוי בהתנהגות הקיימת.


1.2 שגיאת כתיב בקובץ
   בעיה: ServicדCountries.JS (אות ד עברית במקום e).
   פתרון: שינוי שם הקובץ ל-ServicesCountries.js ועדכון כל ה-import/script שמפנים אליו.
   השפעה: קוד נקי יותר, אין השפעה על המשתמש.


1.3 עקביות נתיבים ב-projectLinks
   בעיה: שימוש מעורב – "./pages/", "../pages/", "/pages/".
   פתרון: לאחד לפורמט אחד (למשל /pages/ מכל מקום).
   השפעה: מניעת באגים בנתיבים כשנפתח מעמודים שונים.


1.4 תגית alt ריקה באייקונים
   בעיה: <img src="/icon/..." alt=""> – חסר תיאור לנגישות.
   פתרון: הוספת alt תיאורי: alt="HTML", alt="CSS", alt="בית" וכו'.
   השפעה: שיפור נגישות, SEO. אין שינוי ויזואלי.

חלק ב': שיפורי חוויית משתמש (UX) – ללא שינוי פונקציונליות
═══════════════════════════════════════════════════════════════════════════════

2.1 אנימציית scroll חלקה
   מצב נוכחי: scroll-behavior: smooth כבר קיים ב-html.
   שיפור: לוודא שכל הקישורים הפנימיים (#skis, #shesh וכו') משתמשים ב-hash.
   השפעה: גלילה חלקה כבר קיימת – אין צורך בשינוי.


2.2 משוב ויזואלי לחיפוש
   שיפור: הצגת רשימת תוצאות אפשריות מתחת לשדה החיפוש (autocomplete) בזמן הקלדה.
   מימוש: div נסתר שמתמלא בהתאמות – לחיצה על תוצאה מעבירה לסקשן/עמוד.
   השפעה: חיפוש נוח יותר. החיפוש הקיים ממשיך לעבוד כרגיל.


2.3 "חזרה למעלה" (Back to top)
   שיפור: כפתור צף בתחתית המסך – מופיע אחרי גלילה למטה, לחיצה מחזירה לראש הדף.
   מימוש: כפתור עם position: fixed, display: none עד scroll > 300px.
   השפעה: נוחות בדפים ארוכים. תוספת בלבד.


2.4 טעינת lazy לתמונות ווידאו
   שיפור: loading="lazy" לתמונות שמתחת לקיפול (below the fold).
   מימוש: <img loading="lazy" ...> | <video preload="metadata"> במקום preload="auto".
   השפעה: טעינה מהירה יותר. אין שינוי בתוכן.


2.5 הודעת אישור בטופס צור קשר
   שיפור: אחרי "שלח" – הודעת toast/alert: "תודה! נחזור אליך בהקדם".
   מימוש: preventDefault על submit, הצגת הודעה, אופציונלי – ניקוי השדות.
   השפעה: משוב למשתמש. הטופס עדיין לא שולח לשרת (אין שינוי בתשתית).

חלק ג': נגישות (Accessibility)
═══════════════════════════════════════════════════════════════════════════════

3.1 lang attribute
   שיפור: <html lang="he"> כי רוב התוכן בעברית (כרגיל lang="en").
   השפעה: קוראי מסך וכלי נגישות יזההו נכון את השפה.


3.2 skip link
   שיפור: קישור "דלג לתוכן ראשי" בתחילת הדף – מוסתר ויזואלית, נגיש מקלדת.
   מימוש: <a href="#main" class="skip-link">דלג לתוכן</a> + CSS להסתרה עד focus.
   השפעה: ניווט מהיר למשתמשי מקלדת. תוספת בלבד.


3.3 focus visible
   שיפור: outline ברור על אלמנטים ב-focus (כפתורים, קישורים, שדות).
   מימוש: .nav-button:focus-visible, .search-box:focus-visible { outline: 2px solid... }
   השפעה: ניווט מקלדת ברור יותר. לא משפיע על עכבר.


3.4 aria-labels
   שיפור: aria-label לכפתורים ללא טקסט (למשל אייקוני תפריט נייד).
   מימוש: <a href="#main" aria-label="עמוד הבית">...</a>
   השפעה: קוראי מסך יקראו תיאור ברור. אין שינוי ויזואלי.


3.5 ניגודיות צבעים
   בדיקה: לוודא ניגוד מינימלי 4.5:1 לטקסט רגיל (WCAG AA).
   כלים: Lighthouse, axe DevTools.
   השפעה: קריאות טובה יותר. שינוי צבעים קל אם נדרש.

חלק ד': ביצועים (Performance)
═══════════════════════════════════════════════════════════════════════════════

4.1 דחיסת תמונות
   שיפור: המרה ל-WebP עם fallback ל-JPG/PNG לקבצים גדולים.
   השפעה: טעינה מהירה. אין שינוי בתוכן – רק גודל קובץ.


4.2 וידאו – הפחתת טעינה
   שיפור: poster="..." לתמונת תצוגה מקדימה, preload="metadata".
   השפעה: פחות נתונים בטעינה ראשונית. הווידאו ממשיך לנגן כרגיל.


4.3 CSS – צמצום כפילויות
   שיפור: איחוד כללי חוזרים (#Q1 a, #Q2 a... #Q6 a) ל-class משותף.
   השפעה: קובץ CSS קטן יותר, תחזוקה קלה. אין שינוי בעיצוב.


4.4 defer/async לסקריפטים
   מצב נוכחי: defer כבר קיים ב-Main Menu.js.
   שיפור: לוודא שכל script חיצוני משתמש ב-defer (או async אם מתאים).
   השפעה: טעינת דף מהירה יותר.

חלק ה': SEO ומטא-דאטה
═══════════════════════════════════════════════════════════════════════════════

5.1 meta description
   שיפור: <meta name="description" content="פורטפוליו של מיכאל פפיסמדוב - פיתוח אתרים, פרויקטי HTML ו-JavaScript">
   השפעה: תיאור טוב יותר בתוצאות חיפוש. אין השפעה על הדף עצמו.


5.2 Open Graph / Twitter Cards
   שיפור: meta tags ל-og:title, og:description, og:image לשיתוף ברשתות.
   השפעה: תצוגה יפה כשמשתפים קישור. תוספת בלבד.


5.3 כותרות H1–H6
   בדיקה: מבנה היררכי – H1 אחד, H2 לסקשנים, H3 לכרטיסים.
   השפעה: SEO טוב יותר. אין שינוי ויזואלי אם המבנה כבר נכון.


5.4 canonical URL
   שיפור: <link rel="canonical" href="https://yoursite.com/"> כשהאתר יעלה לאינטרנט.
   השפעה: מניעת כפילות בתוצאות חיפוש.

חלק ו': יכולות חדשות (תוספתיות בלבד)
═══════════════════════════════════════════════════════════════════════════════

6.1 מצב כהה (Dark mode)
   שיפור: toggle להחלפת ערכת צבעים – שמירה ב-localStorage.
   מימוש: CSS variables + class על body, JS למיתוג.
   השפעה: העדפת משתמש. ברירת מחדל נשארת כמו היום.


6.2 אנימציות כניסה (scroll animations)
   שיפור: כרטיסי פרויקטים וכישורים מופיעים ב-fade-in כשנכנסים לתצוגה.
   מימוש: Intersection Observer + class עם animation.
   השפעה: חוויית גלילה נעימה. אין שינוי בתוכן.


6.3 מונה צפיות / GitHub stars
   שיפור: הצגת "כוכבים" מ-GitHub או "צפיות" (אם יש analytics).
   מימוש: GitHub API או Google Analytics.
   השפעה: הוכחת פופולריות. תוספת בלבד.


6.4 קישור ל-LinkedIn / GitHub
   שיפור: אייקונים ב-footer או בכרטיס ביקור – קישור לפרופילים.
   השפעה: קשר ישיר. תוספת בלבד.


6.5 פילטר פרויקטים
   שיפור: כפתורים "הכל | HTML | JS" לסינון כרטיסי הפרויקטים.
   מימוש: JS שמסתיר/מציג לפי data-attribute.
   השפעה: ניווט נוח. כל הפרויקטים נשארים – רק סינון תצוגה.


6.6 גרסה מודפסת
   שיפור: @media print – הסתרת תפריט, וידאו, כפתורים; הדפסת תוכן מרכזי בלבד.
   השפעה: הדפסה נקייה. אין השפעה על תצוגת מסך.

חלק ט': מה לא לעשות (למניעת פגיעה)
═══════════════════════════════════════════════════════════════════════════════

  • לא להסיר פונקציונליות קיימת (חיפוש, כפתור הצג עוד, ניווט).
  • לא לשנות מבנה תיקיות בלי עדכון כל הקישורים.
  • לא להחליף שמות קבצים (HTML) בלי עדכון כל ה-href.
  • לא להוסיף dependencies כבדים (React, jQuery) בלי צורך – האתר קל ומהיר.
  • לא לשנות נתיבי icon/images בלי לוודא שהקבצים קיימים.
  • לא להפעיל form action לשרת בלי backend – יגרום לשגיאה.
